# Quality Checklist for Prompts

Review your prompt before use:

## Task Alignment
- [ ] The prompt directly answers the question or solves the task.
- [ ] No irrelevant information that could distract the model.

## Clarity and Specificity
- [ ] All terms are unambiguous.
- [ ] Constraints are clearly stated (length, format, style).
- [ ] Variables are marked as [variable] or {{variable}}.

## Context and Background
- [ ] Sufficient background is provided for understanding the task.
- [ ] Audience or target group is specified if relevant.
- [ ] Input/output examples are given if needed.

## Output Format
- [ ] Expected format is clearly described (list, table, paragraph, code, etc.).
- [ ] Structural requirements are specified (headings, sections, bullet points).
- [ ] If code is needed, language and style are indicated.

## Style and Tone
- [ ] Desired tone is specified (formal, informal, persuasive, etc.).
- [ ] If brand voice must be followed, it is described.

## Safety and Ethics
- [ ] The prompt does not lead to generation of harmful content.
- [ ] Respect for copyright and confidentiality.
- [ ] No requests for discriminatory or offensive content.

## Testing
- [ ] The prompt has been tested on the selected model.
- [ ] Output quality assessed for relevance and coherence.
- [ ] If necessary, A/B test variants.
